## 👥 Team Overview

BirTics is a team of three engineering students from **Birzeit University, Palestine**, participating in the **WRO Future Engineers 2026** competition.

Each member has a primary responsibility, while major decisions are reviewed by the entire team.
Although responsibilities are divided, subsystem integration and major engineering decisions are discussed collaboratively to ensure consistency across the complete robot.

| Member       | Primary Role                 | Main Focus                                       |
| ------------ | ---------------------------- | ------------------------------------------------ |
| Julnar Assi  | Software Lead                | Perception, algorithms, control, integration     |
| Lana Sayes   | Mechanical Lead              | Chassis, steering, assembly, hardware placement  |
| Nadeen Jaber | Documentation & Testing Lead | Planning, GitHub, documentation, testing records |

```text
Julnar: Software and control
             │
             ▼
        BirTics Robot
             ▲
             │
Lana: Mechanical design

Nadeen: Documentation, planning, and testing
```

The three roles are connected because every software, hardware, and testing decision affects the complete vehicle.

---

## 💻 Julnar Assi — Software Lead

### Role

Julnar leads the design, implementation, and integration of the robot's software.

### Main Responsibilities

| Responsibility           | Description                                                     |
| ------------------------ | --------------------------------------------------------------- |
| **Sensor Processing**    | Reading and interpreting data from the robot's sensors          |
| **Navigation Logic**     | Developing the logic used to follow the track and complete laps |
| **Obstacle Handling**    | Implementing behavior for red and green traffic signs           |
| **Motion Control**       | Generating commands for driving and steering                    |
| **Software Integration** | Combining software modules into one working control system      |
| **Code Testing**         | Debugging, tuning, and validating software behavior             |

### Software Flow

```text
Sensor Input
     │
     ▼
Data Processing
     │
     ▼
Environment Interpretation
     │
     ▼
Decision Logic
     │
     ▼
Motor and Steering Commands
```

### Main Deliverables

* Robot control code
* Sensor-reading modules
* Navigation and obstacle-handling logic
* Motion-control functions
* Software testing results
* Documented and commented source code

## ⚙️ Lana Sayes — Mechanical Lead

### Role

Lana leads the physical design, assembly, and mechanical integration of the vehicle.

### Main Responsibilities

| Responsibility             | Description                                                |
| -------------------------- | ---------------------------------------------------------- |
| **Chassis Design**         | Defining the vehicle's structure, geometry, and dimensions |
| **Steering System**        | Designing and adjusting the steering mechanism             |
| **Mechanical Assembly**    | Building and securing the vehicle components               |
| **Component Placement**    | Positioning motors, sensors, battery, and controller       |
| **Structural Reliability** | Checking balance, strength, clearance, and stability       |
| **Physical Testing**       | Evaluating movement, turning, and mechanical performance   |

### Mechanical Flow

```text
Competition Constraints
        │
        ▼
Chassis Design
        │
        ▼
Steering and Wheel Setup
        │
        ▼
Component Placement
        │
        ▼
Assembly
        │
        ▼
Physical Testing
```

### Main Deliverables

* Chassis design and assembly
* Steering mechanism
* Component-mounting solution
* Mechanical diagrams and photos
* Physical test results
* Records of mechanical changes and improvements

## 🔗 Collaboration

Although each member has a primary role, the team collaborates on decisions that affect more than one subsystem.

| Decision or Task          | Lead            | Support         |
| ------------------------- | --------------- | --------------- |
| Software architecture     | Julnar          | Nadeen          |
| Navigation and control    | Julnar          | Lana            |
| Chassis and steering      | Lana            | Julnar          |
| Sensor placement          | Lana and Julnar | Nadeen          |
| Hardware selection        | All members     | —               |
| Test-plan design          | Nadeen          | Julnar and Lana |
| Test execution            | All members     | —               |
| Engineering documentation | Nadeen          | Julnar and Lana |
| Competition presentation  | Nadeen          | All members     |

### Collaboration Principles

* Each task has one clear lead.
* Major technical decisions are discussed by the whole team.
* Every important change is documented.
* Testing is shared by all members.
* Documentation must match the actual robot.
* Problems and failed tests are recorded, not hidden.
