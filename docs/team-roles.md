### What Julnar Owns

## 🔗 &nbsp;How We Work Together

```
  ┌────────────────────────────────────────────────────────────────┐
  │                                                                │
  │   DECISION TYPE              LEAD              SUPPORTS        │
  │   ─────────────────────────────────────────────────────────   │
  │   Algorithm design           Julnar            Nadeen (doc)    │
  │   Computer vision tuning     Julnar            Lana (test)     │
  │   Chassis geometry           Lana              Julnar (fit)    │
  │   Sensor placement           Lana + Julnar     Nadeen (doc)    │
  │   Test plan design           Nadeen            Lana + Julnar   │
  │   Competition presentation   Nadeen            All             │
  │   Hardware selection         All               —               │
  │   Architecture decisions     All               —               │
  │                                                                │
  └────────────────────────────────────────────────────────────────┘
```

**Our collaboration principles:**

- **One clear lead per domain** — decisions don't stall because ownership is unambiguous
- **All major decisions are documented** before they are implemented — no undocumented pivots
- **Testing is everyone's job** — Nadeen coordinates it, but every member participates in test sessions
- **The journal is updated every session** — if it wasn't written down, it doesn't count
